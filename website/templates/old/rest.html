Welcome to the Cipher RESTful API

The RESTful API provides easy access to formatted information the CIPHER database via a structured URL call. The following guide will demonstrate how to construct a URL for access to varaious database information as well as showcase several examples. The final part of the guide will show how to use POST and PUT functionality to construct complex search queries and add information to the database.

General Collection

The general collection of the database contains basic identifying information of database compounds. This includes compound name, SMILES string, InChi string, and InChi Key. In additition to identifying information, the general collection contains a modification date which denotes the last date and time the compounds information was updated.

The URL format to access this collection of the database with a GET request is as follows:

www.cipher-db.com/rest/general/<input_type>/<input>/<data_type>/<output_type>

Properties Collection

The properties collection of the database contains physical and chemical properties that are both determined experimentally and calculated via computational software. The collection is organzed heierarchically by compound, data source and then property. For example, oxycodone (compound) can have information from the Pubchem database (data sourece) such as octanol-water partition coefficient (property).

The URL format to access this collection of the database with a GET request is as follows:

www.cipher-db.com/rest/properties/<input_type>/<input>/<dataset>/<data_type>/<output_type>
